# ①課題名
飴屋さん販促ツール「じゃんけんアプリ」ブラックバージョンRich

## ②課題内容（どんな作品か）
- 飴屋さんで飴を買ったお客さんにじゃんけんをしてもらう。じゃんけんの勝敗に応じてもらえる飴の数が変わる。
- 初期バージョンでは勝率が3分の1ずつだったが、ブラックバージョンではほぼ勝てない確率に（負け85％、あいこ10％、勝ち5％）　ここまでは5/16の講義で披露
- Richでは、イカサマ演出として、85％の負け枠で負けたときにあおる文言がフラッシュする仕組みを追加

## ⑤工夫した点・こだわった点
- 工夫した点・・・初回のじゃんけんアプリでは基礎機能（プレイヤーのインプットに対し、COMがランダムに返し、プレイヤーとCOMの勝敗を判定する）を作るのに精一杯でしたが、ブラックバージョンでは判定結果を操作したり、Richでは判定結果による表示をさらに複雑にしたり、付加的要素を追加していきました。この課題を通して、基礎機能にどのようなゲーム性ある仕掛けを追加するかを考えることも、アプリ開発のトレーニングになるのかなと思いました。
- こだわった点・・・とにかく黒く、あくどい飴屋さんにすること。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- 難しかった点・・・ゲーム性のある仕掛けのアイデアがなく、全体設計を考えるところに苦労しました。
- 次回トライ・・・基礎機能を作ることで精一杯で、創造性が広がらなかったように思っています。もっと柔軟に考えたいです。
